## 프로젝트 소개
#### 제작 배경
지인들끼리 목돈마련을 위해 곗돈을 모아 관리하는 것을 계모임이라고 합니다. <br>
계모임은 빠른 시일내에 목돈을 마련할 수 있으며 예전에는 계는 우리 사회의 가장 원초적인 '금융회사'였고 '재테크 수단'이었습니다. <br>
허나, 뉴스에도 가끔 등장하는 계주가 곗돈을 들고 잠적하는 최악의 상황이 발생할 수 있습니다. <br>
이러한 상황을 막아주기 곗돈을 투명하게 관리하는 플랫폼을 만들어 ‘계주 리스크’를 줄이겠다는 계획을 구체화하여 프로젝트를 설계하였습니다.

#### 제작 목표
* 누구나 쉽게 계모임으로 목돈 마련하기 위한 UI 제공
* 안전하게 사이트를 이용하기 위해서 중요한 정보 암호화 
* 사용자에게 계모임에 대한 데이터 투명하게 제공
* 수입에 맞지 않는 투자를 방지해서 안전성 보장


## 시작가이드
### Back-End
* Java 11
* Spring Boot 2.7.11
* Spring Security 2.7.2
```
Spring Boot - Tomcat Server Start
```

### Front-End
* React 18.2.0
* npm 8.19.3
* Axios 1.3.6
```
$ cd ./src/main/frontend
$ npm install
$ npm start
```

## 자체 평가 의견
* 먹튀 없는 계모임을 위해서 채권 추심제도를 이용해야했는데 그럴수 없어서 아쉽습니다. 채권 추심제도만 이용할 수 있다면 실무 활용도 가능할 것 같습니다. 
* 리액트를 배워서 컴포넌트를 프로젝트에 활용할 수 있어서 좋았습니다. 
* 프로젝트를 명확한 목표를 설정하고 그 목표를 성공적으로 달성하기 위해 팀원원들 간의 원활한 협업과 팀워크를 진행하였고 의사소통과 역할 분담을 효율적으로 수행했습니다.
* 단일 JWT는 토큰 유출 시 보안적으로 좋지 않음으로 토큰을 주기적으로 갱신하는 Refresh Token을 도입하여 보안을 강화해야 합니다.
* 첨부파일 기능 일부를 구현못하여 게시판 기능 100%를 다 구현을 못한것 같아 아쉬웠고, 2차 기간에는 못했던 조회수 중복방지 기능을 이번 3차 기간에는 구현하게되어 조금은 성장했다 느꼈고, 페이징을 front-end 부분인 react에서 구현했는데 다음번엔 back-end 부분에서 페이징을 구현해보고 싶습니다.
* 새로운 기술인 리액트를 통해 여러가지 기능들이 있는 웹 페이지를 구현할 수 있게 되었습니다.
* 리액트에서 상태 라이브러리(Redux)를 많이 활용하지 못한 부분이 아쉬웠습니다.
